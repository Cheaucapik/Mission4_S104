Mission 2

Contexte


      L'objectif principal de cette mission est d'explorer et d'analyser la base de données de l'entreprise KDou en utilisant une interface low-code, plus précisément le logiciel Microsoft Access. Ce dernier permet de créer et d'exécuter des requêtes à l'aide d'une interface graphique (QBE - Query By Example) qui génère automatiquement le code SQL correspondant. Cela permet une exploitation plus rapide et accessible des données sans nécessiter une maîtrise approfondie du langage SQL.


      Dans cette mission, la base de données KDou contient plusieurs tables reliées entre elles, représentant des informations sur les produits, les fournisseurs, les clients et les commandes. L'analyse des données vise à produire des indicateurs commerciaux et logistiques permettant d'évaluer les performances de l'entreprise.


      Ainsi, l’objectif est de réaliser diverses requêtes afin d’extraire des informations utiles pour l’entreprise, tout en s’assurant de la cohérence et de la qualité des données. Ces requêtes permettront de répondre à un ensemble de questions pratiques concernant les fournisseurs, les produits, les ventes et les clients de l'entreprise.

1er indicateur : Classement des 3 Villes avec la Plus Grande Clientèle

Un classement des 3 premières villes avec le plus de clients est demandé.

Cet indicateur permet de savoir dans quelles villes la clientèle est la plus importante, ce qui peut aider dans la prise de décisions de marketing et de logistique.

Code SQL généré :

SELECT TOP 3 Client.VilleCli, Count(Client.CodeCli) AS CompteDeCodeCli
FROM Client
GROUP BY Client.VilleCli
ORDER BY Count(Client.CodeCli) DESC;

Résultat :

VilleCli CompteDeCodeCli
Agen 3
Paris 3
Nantes 3
Lille 3

2e indicateur : Produits Italiens ou Allemands de 50€ ou Moins

Cet indicateur permet d’identifier les produits de fournisseurs européens (Italie et Allemagne) vendus à un prix attractif (50 € ou moins).
Cela peut aider à proposer des produits européens à prix abordable dans les campagnes de vente.

Code SQL généré :

SELECT Produit.NomProd, Produit.PrixUnit, Fournisseur.Pays
FROM Fournisseur
INNER JOIN Produit ON Fournisseur.NoFour = Produit.NoFour
WHERE (((Produit.PrixUnit)<=50) AND ((Fournisseur.Pays) = "Italie" OR (Fournisseur.Pays) = "Allemagne"))
ORDER BY Fournisseur.Pays;

Résultat :

NomProd PrixUnit Pays
Original Frankfurter grüne Soße 10,83 € Allemagne
Wimmers gute Semmelknödel 27,71 € Allemagne
Mozzarella di Giovanni 29,00 € Italie
Ravioli Angelo 16,25 € Italie

3e indicateur : Chiffre d’Affaires Total Généré

Calculer le chiffre d’affaires total permet de savoir combien de revenus ont été générés à partir des informations de vente.
Cela donne un aperçu de la performance globale des ventes de l’entreprise.

Code SQL généré :

SELECT Sum(PrixUnitCli*QteCom*(1-Remise)) AS [Chiffre d'affaire]
FROM Client
INNER JOIN (Commande INNER JOIN DetailCommande ON Commande.NoCom = DetailCommande.NoCom) ON Client.CodeCli = Commande.CodeCli;

Résultat :

Chiffre d'affaire
1541632,27670534

4e indicateur : Prix Moyen, Minimum et Maximum des Produits

Cet indicateur permet d’analyser les gammes de prix des produits de l’entreprise.
Connaître le prix moyen, ainsi que les prix minimum et maximum, permet d’évaluer l’éventail des prix proposés et d’identifier les extrêmes, ce qui peut influencer les stratégies de prix.

Code SQL généré :

SELECT Min(Produit.PrixUnit) AS [Prix min], Max(Produit.PrixUnit) AS [Prix max], Avg(Produit.PrixUnit) AS [Prix moy]
FROM Produit;

Résultat :

Prix min Prix max Prix moy
2,08 € 320,00 € 33,54 €
