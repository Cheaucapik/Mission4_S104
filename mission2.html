Mission 2

Contexte


      L'objectif principal de cette mission est d'explorer et d'analyser la base de données de l'entreprise KDou en utilisant une interface low-code, plus précisément le logiciel Microsoft Access. Ce dernier permet de créer et d'exécuter des requêtes à l'aide d'une interface graphique (QBE - Query By Example) qui génère automatiquement le code SQL correspondant. Cela permet une exploitation plus rapide et accessible des données sans nécessiter une maîtrise approfondie du langage SQL.


      Dans cette mission, la base de données KDou contient plusieurs tables reliées entre elles, représentant des informations sur les produits, les fournisseurs, les clients et les commandes. L'analyse des données vise à produire des indicateurs commerciaux et logistiques permettant d'évaluer les performances de l'entreprise.


      Ainsi, l’objectif est de réaliser diverses requêtes afin d’extraire des informations utiles pour l’entreprise, tout en s’assurant de la cohérence et de la qualité des données. Ces requêtes permettront de répondre à un ensemble de questions pratiques concernant les fournisseurs, les produits, les ventes et les clients de l'entreprise.

1er indicateur : Classement des 3 Villes avec la Plus Grande Clientèle

Un classement des 3 premières villes avec le plus de clients est demandé.

Cet indicateur permet de savoir dans quelles villes la clientèle est la plus importante, ce qui peut aider dans la prise de décisions de marketing et de logistique.
